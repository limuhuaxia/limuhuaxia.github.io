open stack实验

实验准备：纯净的Linux系统或ubantu系统

open stack官网：docs.openstack.org

选择一个较旧的版本下载，因为新的可能存在BUG未修复

使用ubantu系统安装，版本18.04,必须联网

apt-get update 更新软件

uname -a 查看系统版本

ping baidu.com 检查网络

准备两台虚拟机，两台centos或两台ubantu,可以克隆一台，管理—>克隆—>完整克隆

hostnamectl set-hostname control

hostnamectl set-hostname compute

因为是克隆的，所以修改网络

vim /etc/network/interfaces

此处应有图片

配置对应关系

vim /etc/hosts

此处应有图片

测试两台主机互通性

ip -a

此处应有图片

安装tools，可以不装，不影响

apt -get install net-tools

远程登录

apt-get install openssh-server -y

vim /etc/ssh/sshd_config

图

permitrootlogin：yes

passwordxxxxx：yes

更新源，华为、阿里都可以

时间同步

ntpdate cn.pool.ntp.org

date

更换pip源

pwd

mkdir .pip 创建隐藏文件夹

vim .pip/pip.conf

去阿里的pypi下复制

下载devstack，复制地址，版本ussuri，下载地址：https：docs.openstack.org/devstack/latest/

git clone +地址 -b 切换版本

useradd - s /bin/bash -d /opt/stack -m stack

echo “stack ALL=(ALL) NOPASSWD: ALL” |

vim /loc

下载中报错：https://bootstrap.pypa.io/get-pip.py

准备工作：
如果非第一次安装，需要将之前的devstack环境卸载掉，具体步骤如下：

sudo rm -rf /opt/stack
sudo apt autoremove
sudo apt autoclean

安装工作：
更换apt源为阿里源
cd ~
mv /etc/apt/sources.list /etc/apt/sources.list.bak
sudo gedit /etc/apt/sources.list

在上述命令后，会弹出一个文本填写界面，在界面内填入以下文本：

deb http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse

deb-src http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse

更新源：

sudo apt-get update
sudo apt-get upgrade

设置时区
输入命令：sudo dpkg-reconfigure tzdata后，选择Asia，再选择Shanghai
之后执行如下命令：
sudo apt-get install ntpdate
sudo ntpdate cn.pool.ntp.org
date

更换pip源为豆瓣源
sudo apt-get install python-pip
sudo pip install –upgrade pip
sudo pip install -U os-testr
mkdir .pip
sudo gedit .pip/pip.conf

在新弹出来的文本框中输入如下内容：

[global]
index-url = http://pypi.douban.com/simple/
trusted-host = pypi.douban.com

下载devstack
sudo apt-get install git
git clone https://github.com/openstack-dev/devstack

创建stack用户
执行命令：devstack/tools/create-stack-user.sh

将devstack目录移动至/opt目录下并更名为stack

sudo mv devstack /opt/stack
sudo chown -R stack:stack /opt/stack
sudo chmod -R 777 /opt/stack

切换用户身份
sudo su - stack

创建local.conf配置文件
输入命令：vi local.conf
在vi界面中按下i键
将下面的内容复制进去（注意HOST_IP)为自己的vm ip：
[[local|localrc]]

Define images to be automatically downloaded during the DevStack built process.

DOWNLOAD_DEFAULT_IMAGES=False
IMAGE_URLS=”http://download.cirros-cloud.net/0.3.4/cirros-0.3.4-x86_64-disk.img"

use TryStack git mirror

GIT_BASE=http://git.trystack.cn
NOVNC_REPO=http://git.trystack.cn/kanaka/noVNC.git
SPICE_REPO=http://git.trystack.cn/git/spice/sice-html5.git

Credentials

DATABASE_PASSWORD=admin
ADMIN_PASSWORD=admin
SERVICE_PASSWORD=admin
SERVICE_TOKEN=admin
RABBIT_PASSWORD=admin
#FLAT_INTERFACE=enp0s3

HOST_IP=”your vm ip”

进入files目录下载文件（如果不成功就多执行几次wget命令）
cd files/
wget -c https://github.com/coreos/etcd/releases/download/v3.1.10/etcd-v3.1.10-linux-amd64.tar.gz
wget -c https://github.com/coreos/etcd/releases/download/v3.1.7/etcd-v3.1.7-linux-amd64.tar.gz

回退到上一级目录
cd ..
运行stack.sh文件
./stack.sh

注意事项
遇到超时错误
重新运行./stack.sh
遇到cannot uninstall ‘simplejson’错误
执行命令：sudo pip install simplejson –ignore-installed simplejson后再重新运行./stack
其他错误
其他错误都在重新执行./stack后自动解决了
如果实在不行，一直停留在同一个错误，就先执行./unstack.sh和./clean.sh后再执行./stack.sh
加油啊朋友们，胜利在望了～～～
